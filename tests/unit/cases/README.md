# Unit Test Case Directory

Simplified Chinese version: [README-zh-CN.md](README-zh-CN.md)

## 1. Setup

Before adding or maintaining cases in this directory, make sure the repository root has completed the basic build:

```bash
make cgen-llvm
make runtime-lib
```

If you only maintain expected output files, no additional setup is required. If you want to verify new cases locally, you need at least:

- `app/cgen-llvm`
- `app/parser`
- `app/semant`
- `bin/.i686/lexer`
- `lib/runtime/libruntime.so`

## 2. Tests

### 2.1 Run All Cases

```bash
make test-units
```

### 2.2 Run One Case

```bash
make unit-test example.cl
make unit-test cells.cl
```

### 2.3 Manually Verify One Case

Using `example.cl` as an example:

```bash
cd ../../../app
../bin/.i686/lexer ../tests/unit/cases/example.cl \
  | ./parser ../tests/unit/cases/example.cl \
  | ./semant ../tests/unit/cases/example.cl \
  | ./cgen-llvm ../tests/unit/cases/example.cl \
  > ../tests/unit/cases/example.actual.ll

clang++ ../tests/unit/cases/example.actual.ll \
  -L../lib/runtime -lruntime \
  -Wl,-rpath,../lib/runtime \
  -o ../tests/unit/.artifacts/example

printf 'q\n' | ../tests/unit/.artifacts/example
```

If the program is non-interactive, you can run the generated executable directly without feeding `q\n`.

## 3. Case Format

Each test should contain at least these files:

- `name.cl`: input COOL program
- `name.expected.txt`: expected standard output from the program

After a test run, this file is usually generated as well:

- `name.actual.ll`: LLVM IR generated by `cgen-llvm`

For the `example` case:

- `example.cl`
- `example.actual.ll`
- `example.expected.txt`

## 4. Maintenance Rules

- `.expected.txt` stores only final program output. Do not mix in debug logs.
- `.actual.ll` is a generated artifact. It can be kept for debugging, but its content should be updated by the test workflow instead of maintained by hand long term.
- When adding an interactive program, remember that the runner provides only `q\n` by default. If a program needs more input, adjust the runner command template or verify it manually.
- The `cases/` directory may also keep reference `.cl` files synced from `examples/`. `make test-units` runs only samples with non-empty expectations by default. Empty `.expected.txt` files are placeholders and are not treated as valid golden cases.
