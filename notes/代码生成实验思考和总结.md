# 代码生成模型 

表达式使用栈模型。如a+b+c  先计算表达式a的值，存入栈(由+决定)，再计算表达式b的值，存入栈(由+决定，这里可以不存)。再执行+。那么最后表达式的值在栈中。如果使用寄存器，那么在acc中。

也就是经过一个表达式后，栈不变，新的值放到ACC中，(那么前一个ACC是直接舍弃的)。Todo:验证

在cool语言中，变量定义需要使用let表达式，定义的变量使用范围在let的body里面（还有type语句）。所以其不需要像C语言中关心之前定义的变量，经过不知道多少个表达式后怎么找到其位置的问题(假设表达式产生一个值，cool语言每个表达式都产生一个值)。

{[[expr; ]]+} 所以表达式不需要入栈中间表达式的值

考虑类函数的实现

- expr.func()
  当前的类正处于的类放在a0当中

  对于函数调用代码，在fp压栈后，继续压栈$s0。在计算en-e1后，计算e0，把其结果存放到$s0中，最后跳转

  对于函数定义代码，在$sp指针移动需要多四个单位，同时需要恢复老的s0

- 1.4 特殊init函数
  init函数就是就是执行init表达式子中的值，赋值给SELF寄存器指向的类的具体位置，并且init函数返回值就是这个类。

  在计算init表达式时，最关键的就是变量引用，其他的使用正常的生成代码方式即可。根据操作语法定义init只能引用类中的定义。所以此时代表“栈中数据”的数据结构应该为空。在后面生成代码的时候就用不到其他数据


# 使用stack machine可以防止寄存器被修改
比如下面的代码，直接使用T1和T2会发生错误
```C++
void eq_class::code(ostream &s) {
  /** 我的代码开始 */
  // cool语言判断两个变量是否相等的逻辑是，非基础类地址相等，基础类比较值
  // 不能直接使用t1，和t2.需要存到栈中。因为t1有可能被修改，如true=(1=1)
  e1->code(s);
  emit_push(ACC, s);
  e2->code(s);
  emit_load(T1, 1, SP, s);
  emit_addiu(SP,SP,4,s);
  emit_move(T2, ACC, s);
  // a0为true， a1为false
  emit_load_bool(ACC, BoolConst(1), s);
  emit_load_bool(A1, BoolConst(0), s);
  emit_beq(T1, T2, global_labal ,s);
  emit_jal("equality_test",s);
  emit_label_def(global_labal++,s); 
  /** 我的代码结束 */
}
```


# 几个关键的表达式
## new 表达式示例
``` C++
class Base inherits IO
{
  identify() : Object
  {
    out_string( "base\n" )
  };

  duplicate() : Base
  {
    new SELF_TYPE
  };
};


class Derived inherits Base
{
  identify() : Object
  {
    out_string( "derived\n" )
  };
};


class Main 
{
  main() : Object
  {
    (new Derived).duplicate().identify()
  };
};
```
上面的结果是输出derived。所以new SELF_TYPE是根据执行时的动态类型(此时self的寄存器找到要复制的类)，其他情况则是根据name在代码中写死。

执行完emit_jal("Object.copy",s);后，寄存器a0就是将要执行的类，这和dispath的情况是统一的，这就可以直接跳转init函数，相当于再调用一个普通的init函数进行初始化。

## dispatch表达式
执行参数表达式，压栈，再执行左边的调用表达式。最后查找到跳转的函数，跳转。


## 赋值表达式和Object表达式：
  对于赋值表达式，计算后右边的表达时候最关键的是存放在哪。Object表达式需要知道使用哪一个地址的值。

  cool中类成员变量在堆中，由函数定义、let语句、typecase语句产生的变量在栈中。优先到栈中找，找不到再到堆中查找。
  
  在堆中的变量通过SELF寄存器和偏移查找，在栈中的寄存器有FP(SP会发生变化，使用FP偏移值一直不变)和偏移查找。

  语句可以嵌套，所以当存在定义变量的语句需要改变特定的数据结构，以使得object代码生成时可以找到正确的位置。
